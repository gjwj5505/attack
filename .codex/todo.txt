다음 우선순위: analyzer soundness/crash 수정

1. Crash 제거
- `Itv.filter_*` / `filter_t` / `filter_f`가 `Bot`이나 불가능 interval에서
  터지지 않게 하기
- unreachable branch는 analyzer를 죽이지 말고 `Bot`으로 유지하기

2. Interval normalize
- interval 생성 helper 추가
  - `l <= r`이면 `Itv(l,r)`
  - `l > r`이면 `Bot`
- filter 결과가 invalid interval을 만들지 않게 하기

3. Bound 비교 soundness
- `Itv.bop Lt/Le/Gt/Ge`에서 `Bound.t`를 OCaml polymorphic compare로 비교하지
  않기
- `-∞ < n < +∞` 순서를 명시한 compare/leq helper 사용

4. Abs_mem missing variable semantics 통일
- `find`는 없는 변수를 `[0,0]`으로 본다
- join/widen도 missing variable을 `[0,0]`으로 취급해야 sound함
- 한 branch에서만 `x := 1`이면 join 결과는 `x=[0,1]`이어야 함

5. Bot memory 보존
- `Abs_mem.add Bot` 재검토
- assignment/filter transfer에서 `Bot` state가 되살아나지 않게 하기

6. Regression tests
- branch join에서 unassigned variable 기본값 `[0,0]` 반영
- 불가능 조건이 `Bot`으로 normalize되는지
- ±∞ bound 비교가 sound한지
- Bot state가 assignment/filter로 되살아나지 않는지

나중에 할 일
- C-style syntax로 바꾸기: assign, if-else, while, block, bool condition
- side effect가 있는 expression을 고려할지 결정하기
- component lookup 최적화 검토: env/prog/id 기준 index 또는 component_pool
