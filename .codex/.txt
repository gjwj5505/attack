다음 우선순위: analyzer soundness/crash 수정 계속

완료
- `Itv.filter_*`가 `Bot` 입력에서 crash하지 않고 `Bot`을 보존하게 함
- filter 결과 invalid interval을 `Bot`으로 normalize하는 helper 추가
- `Bound.compare` / `Bound.lt` 추가, `Lt`에서 OCaml polymorphic compare 제거
- `Abs_mem.add Bot` / assignment transfer가 `Bot` state를 되살리지 않게 함
- `x |-> Itv.Bot` 대신 memory 전체를 `Bot`으로 collapse하게 함
- missing variable 기본값 `[0,0]`을 `find`, `leq`, `equal`, `join`, `widen`에 반영
- `Abs_mem.join`과 `Abs_mem.widen`을 분리
- smoke check:
  - `dune build` 통과
  - `dune test` 통과
  - 기존 `examples/*.d` analyzer 실행 통과
  - `examples/5.d` 추가, unreachable branch / implicit `[0,0]` 확인

1. Join/widen 적용 지점 분리
- 현재 analyzer update는 모든 edge에서 `Abs_mem.widen old mem`만 사용한다
- branch merge / ordinary forward edge에서는 `Abs_mem.join`을 쓰고,
  loop back-edge 또는 widening point에서만 `Abs_mem.widen`을 쓰도록 바꾸기
- 기대 예시:
  - 한 branch에서만 `x := 1`이면 merge 결과는 `x=[0,1]`
  - loop에서는 finite convergence가 유지되어야 함

2. Interval arithmetic crash/soundness 점검
- `Bound.add`의 `+∞ + -∞`, `-∞ + +∞` failwith가 analyzer에서 reachable한지 확인
- `Itv.add`, `Itv.mul`, `Itv.neg`의 infinity corner case 점검
- `Itv.bop Eq/Ne/Lt/Le/Gt/Ge` boolean interval 결과 soundness 점검
- `Itv.filter_eq`에 해당하는 현재 `meet` 기반 처리와 `filter_ne` precision/soundness 점검

3. Analyzer regression tests 자동화
- 지금은 `examples/` smoke test만 있으므로 `test/`에 analyzer regression 추가
- 포함할 케이스:
  - branch join에서 unassigned variable 기본값 `[0,0]` 반영
  - 불가능 조건이 `Bot`으로 normalize되는지
  - ±∞ bound 비교가 sound한지
  - Bot state가 assignment/filter로 되살아나지 않는지
  - `[0,0]` explicit binding과 missing key가 같은 의미로 처리되는지

4. Analyzer crash fuzz/synthesis 재시도
- 위 regression이 들어간 뒤 `-attack` synthesis를 다시 돌려 analyzer crash가 재현되는지 확인
- crash가 아니라 top false alarm 합성이 나오는지 확인

나중에 할 일
- C-style syntax로 바꾸기: assign, if-else, while, block, bool condition
- side effect가 있는 expression을 고려할지 결정하기
- component lookup 최적화 검토: env/prog/id 기준 index 또는 component_pool
